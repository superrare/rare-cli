# RARE CLI and SDK State

Generated from the current source in `src/commands/**`, `src/sdk/types.ts`, and the test tree.

Global CLI behavior:

- `rare --json ...` is available globally.
- `--chain <chain>` and `--chain-id <id>` resolve through CLI config unless noted otherwise.
- `--yes` is listed only where the command currently registers it in Commander. Several write commands do not expose `--yes`, so they do not participate in the global pre-action confirmation hook.

Coverage shorthand:

- CLI deterministic: `test/e2e/cli.test.ts`
- CLI live: `test/e2e-live/**`
- SDK unit: `test/unit/sdk/**`, plus command core tests under `test/unit/commands/**`
- SDK integration: `test/integration/sdk/**`
- SDK fork integration: `test/integration/sdk/fork-batch-offer.test.ts`

## Gaps Worth Tracking

- Batch listing CLI has help/wiring coverage and SDK unit coverage, but no live CLI or direct SDK integration write flow was found.
- `liquid-edition set-render-contract` has CLI help coverage but no live write test was found.
- Several collection owner/royalty/metadata write commands do not expose `--yes` even though older docs listed it. The table above reflects the current Commander source.
- Single-token listing, offer, and auction write paths have strong CLI live coverage and SDK core/read coverage, but not many direct SDK write integration tests outside the CLI.
